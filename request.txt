# Meow Rescue - 개선사항 구현 프롬프트

아래 4가지 개선사항을 기존 Meow Rescue 프로젝트에 적용해주세요.
기존 프로젝트의 기술 스택(Kotlin 2.2.0, SurfaceView + Canvas, Room 2.6.1)과 패키지 구조를 유지하면서 구현합니다.

---

## 1. 퍼즐 품질 필터 (PuzzleGenerator.kt 수정)

시드 기반으로 생성된 퍼즐이 항상 재미있다는 보장이 없으므로, 생성 후 품질 검증 단계를 추가합니다.

### 요구사항
- BFS 솔버로 최적 해를 구한 뒤, 아래 조건에 해당하면 해당 시드를 스킵하고 다음 시드로 재생성합니다.
  - **단방향 풀이**: 모든 이동이 한 방향(예: 위로만)으로만 구성된 퍼즐은 제외
  - **단일 블록 풀이**: 고양이 블록 외에 1개 블록만 움직이면 풀리는 퍼즐은 제외 (입문 스테이지 1~5 제외)
  - **독립 이동 풀이**: 블록 간 의존 관계가 전혀 없는(어떤 순서로든 풀리는) 퍼즐은 제외
- 재생성 시 시드를 `baseSeed + offset` 형태로 증가시키되, 최대 50회까지만 시도합니다.
- 50회 안에 통과하는 퍼즐이 없으면 마지막 생성된 퍼즐을 그대로 사용합니다.
- 최종 사용된 시드 오프셋을 캐싱하여 같은 스테이지 재진입 시 동일한 퍼즐이 나오도록 합니다.

---

## 2. 고양이 컬렉션 시스템 (CollectionActivity.kt 구현)

스테이지 진행에 따라 고양이를 해금하는 수집 시스템을 구현합니다.

### 데이터 구조
- 고양이는 총 20종으로 구성하며, 각 고양이는 id, 이름, 해금 조건(필요 스테이지 번호), 해금 여부를 가집니다.
- 해금 조건 예시: 스테이지 1 클리어 → 1번 고양이, 스테이지 10 클리어 → 2번 고양이, 이후 약 7~8 스테이지 간격으로 해금
- Room DB에 CatCollection 엔티티를 추가하고, UserProgress와 연동합니다.

### UI 구성
- 4열 그리드 레이아웃으로 고양이 카드를 표시합니다.
- 해금된 고양이: 컬러 아이콘 + 이름 표시
- 미해금 고양이: 회색 실루엣 + 자물쇠 아이콘 + "Stage XX 클리어 시 해금" 텍스트
- 스테이지 클리어 시 새 고양이가 해금되면 축하 다이얼로그를 표시합니다.

### 게임 연동
- 해금된 고양이 중 하나를 선택하면 게임 내 고양이 블록의 외형이 변경됩니다.
- PuzzleView에서 선택된 고양이에 따라 블록 위에 그려지는 고양이 이미지를 변경합니다.

## 3. 전면 광고 타이밍 변경 (AdManager.kt + PuzzleActivity.kt 수정)

클리어 성취감을 보존하기 위해 광고 노출 시점을 변경합니다.

### 요구사항
- 기존: 스테이지 클리어 직후 전면 광고 표시
- 변경: 클리어 팝업(별점 + 다음 스테이지 버튼) 표시 → 유저가 "다음 스테이지" 버튼 클릭 → 전면 광고 표시 → 광고 종료 후 다음 스테이지 로드
- 매 스테이지마다 광고를 표시하지 않고, 3스테이지마다 1회 전면 광고를 표시합니다.
- AdManager에 스테이지 카운터를 두고, `shouldShowInterstitial()` 메서드로 판단합니다.
- 광고 로드 실패 시 광고 없이 바로 다음 스테이지로 진행합니다 (유저 흐름을 막지 않음).
- 전면 광고는 다음 스테이지 진입 전에 미리 preload 해둡니다.

## 4. 애니메이션 강화 (PuzzleView.kt 수정)

SurfaceView + Canvas 기반으로 아래 애니메이션을 추가합니다.

### 블록 스냅 애니메이션
- 유저가 블록 드래그를 놓으면, 블록이 가장 가까운 격자 위치로 부드럽게 스냅됩니다.
- 보간 함수는 DecelerateInterpolator 스타일(ease-out)을 사용합니다.
- 애니메이션 시간: 100~150ms
- 스냅 중에는 해당 블록에 대한 추가 입력을 무시합니다.

### 고양이 탈출 연출
- 고양이 블록이 출구에 도달하면 아래 순서로 연출합니다:
  1. 출구 방향 벽이 열리는 애니메이션 (200ms)
  2. 고양이 블록이 출구 밖으로 슬라이드 아웃 (300ms, ease-in 가속)
  3. 화면에 별/하트 파티클 이펙트 (500ms)
  4. 클리어 팝업 표시
- 파티클은 Canvas.drawCircle로 간단하게 구현하되, 랜덤 크기/색상/궤적으로 자연스럽게 표현합니다.
- 파티클 색상은 Theme.kt의 파스텔 톤 팔레트를 사용합니다.

### 블록 이동 효과음 연동
- 블록이 스냅될 때 SoundManager의 블록 이동 효과음을 재생합니다.
- 고양이 탈출 시에는 별도의 클리어 효과음을 재생합니다.
