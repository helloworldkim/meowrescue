# Meow Rescue — Game Design Document (GDD) v4.0

> **슬라이딩 블록 퍼즐 게임. 블록을 밀어서 갇힌 고양이를 탈출시키세요!**

---

## 1. 게임 개요 (Game Overview)

| 항목 | 내용 |
|------|------|
| **게임명** | Meow Rescue |
| **장르** | 슬라이딩 블록 퍼즐 (Rush Hour / Unblock Me 스타일) |
| **플랫폼** | Android (Kotlin Native) |
| **타겟 유저** | 전연령, 캐주얼 퍼즐 게임 애호가 |
| **수익 모델** | 광고 기반 (AdMob 배너/전면/보상형 광고) |
| **핵심 한 줄 요약** | 격자판 위의 블록을 밀어서 고양이 블록이 출구로 나갈 수 있도록 길을 만드는 슬라이딩 퍼즐 게임 |

### 1.1 게임 컨셉

플레이어는 격자판 위에 놓인 블록들을 드래그하여 이동시킵니다. 가로 블록은 좌우로, 세로 블록은 상하로만 이동 가능합니다(1칸짜리 블록은 상하좌우 모두 이동 가능). 고양이 블록이 출구까지 도달하면 스테이지 클리어입니다.

모든 스테이지는 시드 기반으로 자동 생성되며, BFS 솔버가 풀이 가능성과 최적 이동 횟수를 검증합니다. 품질 필터가 단조로운 퍼즐을 걸러내어 다양하고 재미있는 퍼즐만 제공합니다.

### 1.2 게임의 차별점

- **다방향 출구**: 고양이가 RIGHT, LEFT, TOP, BOTTOM 네 방향으로 탈출 가능
- **품질 필터**: 단일 방향/단일 블록/독립 이동 퍼즐을 자동 필터링하여 품질 보장
- **고양이 컬렉션**: 20마리 고양이를 스테이지 클리어로 해금하고, 게임에서 사용
- **탈출 애니메이션**: 벽 열림 → 고양이 슬라이드 → 파티클 이펙트의 연출
- **절차적 생성 + BFS 검증**: 모든 스테이지가 풀이 가능하며 최적 해가 보장됨
- **스냅 애니메이션**: 블록 이동 시 부드러운 ease-out 스냅 효과

## 2. 핵심 게임플레이 (Core Gameplay)

### 2.1 기본 메커니즘

#### 2.1.1 블록 이동

- **격자**: 5x5 ~ 7x7 크기 (스테이지에 따라 증가)
- **블록 종류**: 고양이 블록(1칸), 열쇠 블록(1칸, 금색), 일반 블록(1~3칸, 가로/세로)
- **이동 규칙**:
  - 가로 블록: 좌우로만 이동
  - 세로 블록: 상하로만 이동
  - **1칸 블록: 상하좌우 모두 이동 가능** (양방향 이동)
- **조작**: 블록을 터치하여 드래그, 놓으면 가장 가까운 칸으로 스냅

#### 2.1.2 출구 방향

스테이지마다 고양이의 탈출 방향이 다릅니다:

| 출구 방향 | 고양이 블록 | 설명 |
|----------|-----------|------|
| **RIGHT** | 1칸 | 오른쪽 벽으로 탈출 |
| **LEFT** | 1칸 | 왼쪽 벽으로 탈출 |
| **TOP** | 1칸 | 위쪽 벽으로 탈출 |
| **BOTTOM** | 1칸 | 아래쪽 벽으로 탈출 |

고양이 블록이 1x1이므로 출구와 다른 행/열에서 시작하여 **경로 탐색** 퍼즐이 가능합니다. 출구 방향은 시드 기반으로 결정되며, 100개 스테이지에서 네 방향이 골고루 등장합니다.

#### 2.1.3 열쇠-자물쇠 메커니즘 (스테이지 16+)

| 요소 | 설명 |
|------|------|
| **열쇠 블록** | 금색 1x1 블록, 드래그로 이동 가능 |
| **자물쇠 셀** | 출구 옆 고정 위치, 금색 테두리 표시 |
| **작동 방식** | 열쇠 블록을 자물쇠 셀 위로 이동해야 출구가 열림 |
| **출구 표시** | 열쇠 미배치 시 출구 화살표가 흐리게 표시 (alpha 80) |

#### 2.1.4 체크포인트 메커니즘 (스테이지 31+)

| 요소 | 설명 |
|------|------|
| **체크포인트 셀** | 별(⭐) 마커가 표시된 셀 |
| **배치 규칙** | 고양이~출구 직선 경로에서 벗어난 위치 (우회 강제) |
| **작동 방식** | 고양이가 체크포인트를 경유(pass-through 포함)해야 탈출 가능 |
| **시각 피드백** | 미도달=회색(alpha 60), 도달=금색(alpha 220) + 400ms 펄스 링 |

#### 2.1.5 스테이지별 피처 적용

| 구간 | 피처 |
|------|------|
| **1~15** | 1x1 고양이만 (새 이동 방식 학습) |
| **16~30** | 열쇠 50% 확률 등장 |
| **31~50** | 열쇠 또는 체크포인트 (50/50, 상호 배타) |
| **51+** | 열쇠 + 체크포인트 동시 가능 |

#### 2.1.6 클리어 조건

고양이 블록이 출구까지 이동하면 클리어됩니다. 추가 조건이 있는 경우 모두 충족해야 합니다:
- **열쇠-자물쇠**: 열쇠 블록이 자물쇠 셀 위에 있어야 함
- **체크포인트**: 고양이가 체크포인트 셀을 경유했어야 함

이동 횟수에 따라 별점이 부여됩니다:

| 별점 | 조건 |
|------|------|
| ★★★ | 최적 이동 횟수 이내 |
| ★★ | 최적 × 1.5 이내 |
| ★ | 클리어만 하면 |

### 2.2 게임 플로우

```
┌──────────────────────────┐
│   스테이지 로딩           │
│   (퍼즐 생성 + BFS 솔버) │
└──────────────────────────┘
           ↓
┌──────────────────────────┐
│   PLAYING 상태            │
│   블록 드래그 → 스냅      │
│   Undo / Reset 가능      │
└──────────────────────────┘
           ↓
     고양이 탈출 조건?
      ↙        ↘
    NO          YES
     ↓            ↓
  계속 플레이  ┌──────────────────────────┐
              │   ESCAPING 상태           │
              │   Phase 1: 벽 열림 (200ms)│
              │   Phase 2: 슬라이드 (300ms)│
              │   Phase 3: 파티클 (500ms) │
              └──────────────────────────┘
                       ↓
              ┌──────────────────────────┐
              │   SOLVED (승리) 오버레이  │
              │   별점 + 이동 횟수 표시   │
              │   "Next Stage" 버튼       │
              └──────────────────────────┘
                       ↓
              ┌──────────────────────────┐
              │   Next Stage 클릭         │
              │   (조건 충족 시 전면 광고) │
              │   → 다음 스테이지 로드    │
              └──────────────────────────┘
```

### 2.3 조작 방식

| 조작 | 동작 |
|------|------|
| **블록 드래그** | 블록을 터치 후 이동 축 방향으로 드래그 |
| **스냅** | 드래그 종료 시 가장 가까운 칸으로 120ms ease-out 애니메이션 |
| **Undo** | 마지막 이동을 되돌림 |
| **Reset** | 퍼즐을 초기 상태로 리셋 |
| **일시정지** | HUD의 일시정지 버튼 → Resume / Restart / Quit |

## 3. 퍼즐 생성 시스템 (Puzzle Generation)

### 3.1 의존성 체인 빌더

1. **고양이 배치**: 출구 방향에 따라 고양이 블록 배치
2. **직접 차단**: 고양이 경로 위에 블록 배치 (직접 차단)
3. **간접 차단**: 직접 차단 블록의 이동 경로를 가로막는 블록 배치
4. **중첩 의존성**: 간접 차단 블록을 또 다른 블록이 차단 → 연쇄 의존
5. **BFS 검증**: 최소 이동 횟수가 난이도 임계값 이상인지 확인

### 3.2 BFS 솔버

- 상태를 IntArray(블록 위치 + 체크포인트 비트)로 표현 → PuzzleGrid clone() 없이 탐색
- FNV-1a 64비트 해시로 방문 체크 (충돌 확률 최소화)
- 최대 80,000 상태 (체크포인트 시 120,000), 깊이 35로 제한
- `isSolvedState()`가 exitRow/exitCol + 열쇠 위치 + 체크포인트 도달 상태를 종합 검증
- 체크포인트 경유(pass-through) 감지: 이동 경로 중간에 체크포인트가 있으면 도달 처리

### 3.3 품질 필터 (Quality Filter)

`isQualityPuzzle()` 메서드가 생성된 퍼즐의 품질을 검사합니다:

| 검사 항목 | 조건 | 설명 |
|----------|------|------|
| **단일 방향** | 직접 차단 블록이 모두 같은 방향 | 다양성 부족 → 거부 |
| **단일 블록** | 직접 차단 블록이 1개 이하 (스테이지 > 5) | 너무 쉬움 → 거부 |
| **독립 이동** | 2개 이상 차단 블록이 모두 자유롭게 이동 가능 | 상호 의존성 없음 → 거부 |

- 품질 실패 시 `baseSeed + offset * 10000`으로 재생성 (최대 50회)
- `seedOffsetCache`에 오프셋 저장으로 동일 시드 재진입 시 결정적 결과 보장
- 스테이지 1~5는 입문용으로 단일 블록 검사 면제

## 4. 난이도 설계 (Difficulty Design)

### 4.1 구간별 난이도

| 구간 | 스테이지 | 격자 크기 | 블록 수 | 최소 이동 |
|------|---------|----------|--------|----------|
| **입문** | 1~10 | 5x5 | 3~5 | 2~5 |
| **초급** | 11~30 | 5x5 | 4~7 | 3~7 |
| **중급** | 31~60 | 6x6 | 5~9 | 6~10 |
| **상급** | 61~100 | 6x6 | 6~10 | 11~16 |
| **고급** | 101~150 | 7x7 | 7~12 | 17~20 |
| **마스터** | 151+ | 7x7 | 8~14 | 21~22 |

### 4.2 시드 기반 재현성

- 각 스테이지 번호가 시드로 사용 (`java.util.Random(stage)`)
- 동일 스테이지는 항상 동일한 퍼즐 생성
- 품질 필터 오프셋도 캐시되어 결정적 재진입 보장

## 5. 고양이 컬렉션 시스템 (Cat Collection)

### 5.1 고양이 목록

총 20마리의 고양이가 특정 스테이지 클리어 시 해금됩니다:

| # | 이름 | 해금 스테이지 | # | 이름 | 해금 스테이지 |
|---|------|-------------|---|------|-------------|
| 1 | 나비 | 1 | 11 | 가을 | 50 |
| 2 | 치즈 | 5 | 12 | 겨울 | 55 |
| 3 | 모모 | 10 | 13 | 솜이 | 60 |
| 4 | 구름 | 15 | 14 | 꽃이 | 65 |
| 5 | 호랑 | 20 | 15 | 하늘 | 70 |
| 6 | 까미 | 25 | 16 | 바다 | 75 |
| 7 | 별이 | 30 | 17 | 무지개 | 80 |
| 8 | 달이 | 35 | 18 | 보석 | 85 |
| 9 | 봄이 | 40 | 19 | 왕자 | 90 |
| 10 | 여름 | 45 | 20 | 공주 | 95 |

### 5.2 컬렉션 화면

- **4열 그리드** 형태로 20마리 표시
- **해금된 고양이**: 컬러 이미지 + 이름 표시, 탭하여 선택 가능
- **잠긴 고양이**: 그레이스케일 + "Stage XX 클리어 시 해금" 텍스트
- **선택된 고양이**: 코랄색 테두리 + "SELECTED" 배지

### 5.3 게임 연동

- 선택한 고양이가 퍼즐 화면의 고양이 블록에 표시됨
- 스테이지 최초 클리어 시 새 고양이 해금 → 축하 다이얼로그 표시
- 선택 상태는 SharedPreferences에 영구 저장

## 6. 애니메이션 시스템 (Animation)

### 6.1 블록 스냅 애니메이션

| 항목 | 값 |
|------|-----|
| 지속 시간 | 120ms |
| 보간 함수 | Ease-out Quadratic: `1 - (1-t)²` |
| 효과음 | `playBlockMatch()` |
| 입력 차단 | `snapAnimating` 플래그로 애니메이션 중 입력 무시 |

### 6.2 고양이 탈출 시퀀스

클리어 조건 달성 시 3단계 연출이 진행됩니다:

| Phase | 내용 | 지속 시간 | 효과음 |
|-------|------|----------|--------|
| **1: 벽 열림** | 출구 벽이 양쪽으로 갈라짐 | 200ms | - |
| **2: 슬라이드** | 고양이가 출구 방향으로 미끄러져 나감 | 300ms (ease-in: `t²`) | `playCatRescue()` |
| **3: 파티클** | 파스텔 색상 파티클 폭발 효과 | 500ms | `playLevelClear()` |

### 6.3 파티클 시스템

- 30개의 원형 파티클 생성 (`Canvas.drawCircle`)
- 랜덤 크기(3~8dp), 속도, 방향
- 중력 가속도 적용 (매 프레임 vy += 0.3)
- 알파값 감소로 페이드아웃
- 6가지 파스텔 컬러: 핑크, 블루, 그린, 옐로우, 퍼플, 오렌지

## 7. 수익화 전략 (Monetization)

### 7.1 광고 정책

| 항목 | 값 |
|------|-----|
| **광고 미노출 구간** | 스테이지 1~5 (신규 유저 보호) |
| **전면 광고 간격** | 3 스테이지마다 |
| **세션당 최대 광고** | 10회 |
| **전면 광고 타이밍** | "Next Stage" 버튼 클릭 시 (클리어 직후가 아님) |

### 7.2 광고 유형

| 광고 유형 | 위치 | 설명 |
|-----------|------|------|
| **배너 광고** | 게임 화면 하단 | 상시 표시 |
| **전면 광고** | Next Stage 클릭 시 | 3스테이지 간격, 스테이지 5 이후 |
| **보상형 광고** | 힌트/부활 | 사용자 선택적 (강제 시청 없음) |

### 7.3 광고 플로우

```
스테이지 클리어 → 탈출 애니메이션 → 승리 오버레이 표시
  → 사용자가 "Next Stage" 버튼 클릭
    → shouldShowInterstitial() 체크
      → 조건 충족: 전면 광고 표시 → 광고 닫힘 → 다음 스테이지 로드
      → 조건 미충족: 바로 다음 스테이지 로드
```

### 8.3 핵심 데이터 구조

```kotlin
// ── 퍼즐 블록 ──
data class PuzzleBlock(
    val id: Int, val row: Int, val col: Int,
    val length: Int, val isHorizontal: Boolean,
    val isCat: Boolean = false,
    val isKey: Boolean = false          // 열쇠 블록 여부
)

// ── 퍼즐 격자 ──
class PuzzleGrid(
    val rows: Int, val cols: Int,
    val exitRow: Int, val exitCol: Int,
    val exitDirection: ExitDirection,
    val hasKeyLock: Boolean = false,     // 열쇠-자물쇠 메커니즘 활성화
    val lockRow: Int = -1,              // 자물쇠 셀 행
    val lockCol: Int = -1,              // 자물쇠 셀 열
    val checkpointRow: Int = -1,        // 체크포인트 셀 행
    val checkpointCol: Int = -1         // 체크포인트 셀 열
)

// ── 출구 방향 ──
enum class ExitDirection { RIGHT, LEFT, TOP, BOTTOM }

// ── 이동 기록 ──
data class MoveRecord(
    val blockId: Int, val steps: Int,
    val horizontal: Boolean,
    val setCheckpoint: Boolean = false   // 이 이동이 체크포인트를 활성화했는지 (undo 복원용)
)

// ── 스테이지 피처 ──
data class StageFeatures(
    val hasKey: Boolean,                // 열쇠-자물쇠 활성화
    val hasCheckpoint: Boolean          // 체크포인트 활성화
)

// ── 고양이 정의 ──
data class CatDefinition(
    val id: Int, val name: String,
    val requiredStage: Int, val drawableRes: Int
)
```
